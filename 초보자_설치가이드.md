# 공인중개사 CRM 초보자 설치 가이드

## 기존 골리 트래커와의 관계
- 기존 골리 트래커 저장소: `goalie-tracker`
- 기존 주소: `https://heetae333333-create.github.io/goalie-tracker/`
- 새 CRM 저장소 권장 이름: `realestate-crm`
- 새 CRM 예상 주소: `https://heetae333333-create.github.io/realestate-crm/`
- 서로 다른 저장소와 주소이므로 골리 트래커는 삭제되거나 변경되지 않습니다.

## 준비물
1. GitHub 계정
2. Supabase 계정
3. 이 압축파일
4. 관리자용 이메일 주소와 비밀번호

## 1단계: 압축 풀기
`realestate-crm-mvp.zip` 파일을 마우스 오른쪽 버튼으로 눌러 `압축 풀기`를 선택합니다.
폴더 안에 아래 5개 파일이 있어야 합니다.
- index.html
- styles.css
- app.js
- schema.sql
- README.md

## 2단계: Supabase 프로젝트 만들기
1. 브라우저에서 Supabase 사이트에 접속합니다.
2. GitHub 또는 이메일로 로그인합니다.
3. `New project`를 누릅니다.
4. Organization을 선택합니다.
5. Project name에 `realestate-crm`을 입력합니다.
6. Database Password는 새 비밀번호를 만들어 안전하게 보관합니다.
7. Region은 한국과 가까운 지역을 선택합니다.
8. `Create new project`를 누릅니다.

## 3단계: 데이터베이스 설치
1. Supabase 왼쪽 메뉴에서 `SQL Editor`를 누릅니다.
2. `New query`를 누릅니다.
3. 압축을 푼 폴더의 `schema.sql`을 메모장으로 엽니다.
4. 전체 선택(Ctrl+A), 복사(Ctrl+C)합니다.
5. SQL Editor에 붙여넣기(Ctrl+V)합니다.
6. 오른쪽 아래 또는 상단의 `Run`을 누릅니다.
7. 오류 없이 완료되면 데이터베이스 설치가 끝납니다.

주의: 같은 schema.sql을 두 번 실행하면 이미 존재한다는 오류가 날 수 있습니다. 최초 한 번만 실행합니다.

## 4단계: 이메일 가입 설정
1. Supabase 왼쪽 메뉴에서 `Authentication`을 누릅니다.
2. `Providers` 또는 `Sign In / Providers`로 들어갑니다.
3. Email이 켜져 있는지 확인합니다.
4. 처음 시험할 때는 `Confirm email`을 꺼두는 것을 권장합니다.
   - 켜두면 가입자가 이메일 인증 링크를 눌러야 합니다.

## 5단계: Supabase 연결정보 복사
1. Supabase 왼쪽 아래 `Project Settings`를 누릅니다.
2. `API` 또는 `Data API` 메뉴로 들어갑니다.
3. 아래 두 값을 각각 복사해 메모장에 보관합니다.
   - Project URL: `https://xxxxx.supabase.co` 형태
   - anon public key 또는 Publishable key: 긴 영문 문자열
4. `service_role` 또는 Secret key는 절대 앱에 입력하거나 다른 사람에게 보내지 않습니다.

## 6단계: GitHub에 새 저장소 만들기
1. GitHub에 로그인합니다.
2. 오른쪽 위 `+` 버튼을 누르고 `New repository`를 선택합니다.
3. Repository name에 `realestate-crm`을 입력합니다.
4. Public을 선택합니다.
5. README 추가 옵션은 체크하지 않아도 됩니다.
6. `Create repository`를 누릅니다.

중요: 기존 `goalie-tracker` 저장소를 열어서 파일을 올리면 안 됩니다. 반드시 새 `realestate-crm` 저장소를 만듭니다.

## 7단계: CRM 파일 업로드
1. 새로 만든 `realestate-crm` 저장소 화면에서 `uploading an existing file` 또는 `Add file > Upload files`를 누릅니다.
2. 압축파일 자체를 올리지 말고, 압축을 푼 폴더 안의 아래 파일들을 선택해 올립니다.
   - index.html
   - styles.css
   - app.js
   - schema.sql
   - README.md
   - 초보자_설치가이드.md
3. 아래 `Commit changes`를 누릅니다.

## 8단계: GitHub Pages 켜기
1. `realestate-crm` 저장소의 `Settings`를 누릅니다.
2. 왼쪽 메뉴에서 `Pages`를 누릅니다.
3. Build and deployment의 Source를 `Deploy from a branch`로 선택합니다.
4. Branch는 `main`, 폴더는 `/ (root)`를 선택합니다.
5. `Save`를 누릅니다.
6. 배포 후 주소는 보통 아래와 같습니다.
   `https://heetae333333-create.github.io/realestate-crm/`

## 9단계: 앱과 Supabase 연결
1. 새 CRM 주소를 엽니다.
2. `초기 연결 설정` 화면에서 Project URL을 붙여넣습니다.
3. anon public key 또는 Publishable key를 붙여넣습니다.
4. `저장하고 시작`을 누릅니다.

이 연결정보는 현재 브라우저에 저장됩니다. 다른 PC나 휴대전화에서 처음 접속하면 같은 URL과 공개 키를 한 번 더 입력해야 합니다.

## 10단계: 최초 관리자 계정 만들기
1. CRM 로그인 화면에서 `가입 신청`을 누릅니다.
2. 본인의 이름, 사무소명, 연락처, 관리자용 이메일, 비밀번호를 입력합니다.
3. 가입 후 승인 대기 화면이 나오는 것이 정상입니다.
4. Supabase로 돌아가 `SQL Editor > New query`를 엽니다.
5. 아래 문장에서 이메일만 실제 가입 이메일로 바꿉니다.

```sql
update public.profiles
set role='admin', status='approved'
where email='내가가입한이메일@example.com';
```

6. `Run`을 누릅니다.
7. CRM에서 로그아웃 후 다시 로그인하거나 승인 상태 새로고침을 누릅니다.
8. 관리자 메뉴가 보이면 성공입니다.

## 11단계: 다른 중개사 가입과 승인
1. 다른 중개사가 CRM 주소에 접속합니다.
2. 처음 접속하는 기기라면 Supabase URL과 공개 키를 입력합니다.
3. 가입 신청을 합니다.
4. 관리자가 로그인합니다.
5. `회원 승인·관리` 메뉴에서 해당 중개사를 승인합니다.
6. 승인된 중개사가 다시 로그인하면 사용할 수 있습니다.

## 12단계: 실제 사용 흐름
- 중개사는 `내 고객`에서 자기 고객만 등록하고 관리합니다.
- 중개사는 `내 매물 시트`에서 매물을 등록합니다.
- 공개 매물은 자동으로 `공동매물망`에 나타납니다.
- 모든 승인 중개사는 공동매물망을 검색하고 열람할 수 있습니다.
- 매물 수정·삭제는 현재 담당자 또는 관리자만 가능합니다.
- 퇴사자가 생기면 관리자가 `퇴사자 일괄 이관`에서 새 담당자를 지정합니다.

## 골리 트래커 유지 방법
골리 트래커는 아래 상태로 그대로 둡니다.
- 저장소 이름을 바꾸지 않습니다.
- 저장소를 삭제하지 않습니다.
- 골리 트래커 저장소의 Settings > Pages 설정을 변경하지 않습니다.
- CRM 파일을 골리 트래커 저장소에 업로드하지 않습니다.

두 앱은 다음처럼 동시에 운영됩니다.
- 골리 트래커: `https://heetae333333-create.github.io/goalie-tracker/`
- 공인중개사 CRM: `https://heetae333333-create.github.io/realestate-crm/`

## 꼭 알아둘 점
- 골리 트래커는 오프라인 사용이 가능하지만, CRM은 여러 사람이 같은 데이터를 사용하므로 인터넷 연결이 필요합니다.
- 고객 개인정보가 들어가므로 테스트용 가짜 고객으로 충분히 시험한 후 실제 자료를 입력합니다.
- Supabase Secret key/service_role key는 절대 공개 저장소에 넣지 않습니다.
- 앱 설정에는 Project URL과 anon public key/Publishable key만 사용합니다.
