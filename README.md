# 공인중개사 공동매물 CRM MVP

## 구현된 기능
- 이메일 회원가입 및 관리자 승인 전 접근 차단
- 중개사별 고객 정보 완전 분리
- 중개사 개인 매물 시트
- 개인 시트 등록 시 공동매물망 자동 노출
- 전체 매물 검색 및 거래유형·매물유형·상태·가격 필터
- 매물별 현재 담당 중개사와 사무소 표시
- 본인 매물만 수정·삭제 가능, 관리자는 전체 관리 가능
- 관리자 회원 승인·거절·정지
- 퇴사자의 고객과 매물 전체를 다른 중개사에게 일괄 이관
- 최초 담당자 보존 및 이관 로그 기록
- 모바일 반응형 화면

## 설치 방법
1. Supabase에서 새 프로젝트를 만듭니다.
2. SQL Editor에서 `schema.sql` 전체를 실행합니다.
3. Authentication > Providers에서 Email 로그인을 활성화합니다.
4. 앱에서 회원가입한 뒤 SQL Editor에서 아래 문장을 실제 관리자 이메일로 바꿔 실행합니다.

```sql
update public.profiles
set role='admin', status='approved'
where email='관리자이메일@example.com';
```

5. `index.html`을 열고 Supabase Project URL과 anon public key를 입력합니다.
6. 실제 배포는 폴더 전체를 Netlify, Vercel 또는 GitHub Pages에 업로드하면 됩니다.

## 보안 구조
- 고객: 담당 중개사와 관리자만 읽기 가능
- 공개 매물: 승인된 전체 중개사가 읽기 가능
- 매물 수정: 현재 담당자와 관리자만 가능
- 이관: 관리자 RPC 함수만 실행 가능

## 주의
- 실제 서비스 전 개인정보 처리방침, 이용약관, 접속기록, 백업정책을 추가해야 합니다.
- Supabase 이메일 확인 기능을 켜면 가입자가 이메일 인증까지 완료해야 로그인됩니다.
